# 按分類彙整

## 思考框架型
- cso（gstack）: 以資安長視角做基礎設施、供應鏈與 AI 風險審計的安全評審技能。
- design-consultation（gstack）: 先理解產品再研究競品並提出完整設計系統方案的設計顧問技能。
- design-shotgun（gstack）: 一次產生多個設計變體、比較並收斂偏好的設計探索技能。
- investigate（gstack）: 以根因調查、模式分析、假設驗證到實作的流程來系統化除錯。
- office-hours（gstack）: 用 YC office hours 與 design partner 心法逼出真需求與更強產品方向的策略技能。
- plan-ceo-review（gstack）: 用 CEO 或創辦人視角重新審視計畫範圍、願景與 10 星產品機會。
- plan-design-review（gstack）: 從設計師角度逐維度評估計畫，指出把體驗做成 10 分的方法。
- plan-eng-review（gstack）: 以工程主管視角鎖定架構、資料流、邊界條件與測試覆蓋的計畫評審技能。
- retro（gstack）: 分析提交歷史、工作模式與品質指標，形成週期性工程 retrospective。
- review（gstack）: 在合併前對 diff 做結構性風險審查並盡可能先修問題的 PR review 技能。
- brainstorming（superpowers）: 在任何創作或功能實作前先釐清意圖、需求與設計的前置腦暴技能。
- receiving-code-review（superpowers）: 收到 code review 後先驗證、再回應與實作，而不是表演式同意的技能。
- systematic-debugging（superpowers）: 遇到 bug 時先找根因、再提修復方案的系統化除錯技能。
- test-driven-development（superpowers）: 在寫實作前先寫失敗測試，再按 red-green-refactor 前進的 TDD 技能。
- verification-before-completion（superpowers）: 在宣稱完成前先跑驗證並以證據支撐結論的收尾技能。
- writing-plans（superpowers）: 在動手前先把多步驟工作拆成可執行計畫的規劃技能。
- writing-skills（superpowers）: 建立、測試與維護技能文件本身的技能設計技能。

## 審查型
- autoplan（gstack）: 自動串接 CEO、設計與工程評審，按原則連續審核方案的總控技能。
- benchmark（gstack）: 以瀏覽守護程序建立與比較效能基線的性能檢測技能。
- canary（gstack）: 在部署後持續監控線上站點錯誤、效能與畫面異常的金絲雀檢查技能。
- codex（gstack）: 把 OpenAI Codex 以 review、challenge、consult 三種模式接入工作流的包裝技能。
- cso（gstack）: 以資安長視角做基礎設施、供應鏈與 AI 風險審計的安全評審技能。
- design-review（gstack）: 以設計師視角檢查並修正 UI 一致性、節奏與互動品質的設計 QA 技能。
- land-and-deploy（gstack）: 接手 PR 合併、等待部署並做生產環境驗證的落地部署技能。
- plan-ceo-review（gstack）: 用 CEO 或創辦人視角重新審視計畫範圍、願景與 10 星產品機會。
- plan-design-review（gstack）: 從設計師角度逐維度評估計畫，指出把體驗做成 10 分的方法。
- plan-eng-review（gstack）: 以工程主管視角鎖定架構、資料流、邊界條件與測試覆蓋的計畫評審技能。
- qa-only（gstack）: 只做網站 QA 測試與結構化報告、不直接改碼的報告型 QA 技能。
- qa（gstack）: 系統化測試網站後直接修 bug、逐項驗證並提交原子修復的 QA 技能。
- retro（gstack）: 分析提交歷史、工作模式與品質指標，形成週期性工程 retrospective。
- review（gstack）: 在合併前對 diff 做結構性風險審查並盡可能先修問題的 PR review 技能。
- ship（gstack）: 從合併 base、跑測試、更新版本到推分支建 PR 的完整出貨技能。
- receiving-code-review（superpowers）: 收到 code review 後先驗證、再回應與實作，而不是表演式同意的技能。
- verification-before-completion（superpowers）: 在宣稱完成前先跑驗證並以證據支撐結論的收尾技能。

## 工具程序型
- AGENTS（gstack）: gstack 倉庫內的技能總覽、建置命令與協作慣例說明。
- CLAUDE（gstack）: gstack 倉庫的工程規範、文件生成與提交流程總指引。
- README（gstack）: gstack 的安裝、工作流與核心理念入口文件。
- SKILL（gstack）: 用無頭瀏覽器進行網站 QA、互動驗證與證據擷取的基礎技能。
- benchmark（gstack）: 以瀏覽守護程序建立與比較效能基線的性能檢測技能。
- browse（gstack）: 以無頭瀏覽器快速操作頁面、驗證狀態與蒐集 QA 證據的工具技能。
- canary（gstack）: 在部署後持續監控線上站點錯誤、效能與畫面異常的金絲雀檢查技能。
- careful（gstack）: 為破壞性命令加上事前警告與人工覆核的安全護欄技能。
- codex（gstack）: 把 OpenAI Codex 以 review、challenge、consult 三種模式接入工作流的包裝技能。
- connect-chrome（gstack）: 連接可視化的真實 Chrome 與側邊欄，讓使用者即時觀看瀏覽操作。
- document-release（gstack）: 在發版後同步更新 README、架構與變更記錄等文件的文件收尾技能。
- freeze（gstack）: 把本次會話的可編輯範圍鎖定在指定目錄的編輯邊界技能。
- gstack-upgrade（gstack）: 將 gstack 升級到最新版本並處理升級提示的維護技能。
- guard（gstack）: 同時啟用破壞性命令警告與目錄凍結的全量安全模式技能。
- land-and-deploy（gstack）: 接手 PR 合併、等待部署並做生產環境驗證的落地部署技能。
- learn（gstack）: 管理專案長期 learnings，支援查詢、修剪與匯出的知識累積技能。
- qa（gstack）: 系統化測試網站後直接修 bug、逐項驗證並提交原子修復的 QA 技能。
- setup-browser-cookies（gstack）: 把真實 Chromium 的 cookies 匯入 headless 瀏覽 session 的登入前置技能。
- setup-deploy（gstack）: 偵測部署平台與健康檢查資訊，為 land-and-deploy 建立部署設定。
- ship（gstack）: 從合併 base、跑測試、更新版本到推分支建 PR 的完整出貨技能。
- unfreeze（gstack）: 解除 freeze 設定，重新允許跨目錄編輯。
- dispatching-parallel-agents（superpowers）: 把彼此獨立的任務拆給多個 agent 並行處理的調度技能。
- executing-plans（superpowers）: 依既有實作計畫逐步施工、檢查與回報進度的執行技能。
- finishing-a-development-branch（superpowers）: 在開發完成後結構化決定合併、提 PR 或清理分支的收尾技能。
- requesting-code-review（superpowers）: 在完成主要工作後有條理地請求 code review 的協作技能。
- subagent-driven-development（superpowers）: 在同一會話內用子代理執行獨立實作子任務的開發技能。
- using-git-worktrees（superpowers）: 用 git worktree 建立隔離工作目錄，安全展開多線開發。
