# Skill 人格全量抽取規格

> 本文件是 Codex / Claude Code 的施工指令。
> 目的：從 gstack 和 superpowers 兩個 repo 中，完整抽取所有 skill 的人格、思考框架、方法論。
> 本輪只做抽取和分類，不做合併、瘦身、刪減。產出的是 raw data。

---

## 第一步：下載 repo

```bash
mkdir -p skill-extraction/repos
cd skill-extraction/repos

git clone https://github.com/garrytan/gstack
git clone https://github.com/obra/superpowers
```

---

## 第二步：掃描所有 SKILL.md / CLAUDE.md / AGENTS.md

掃描兩個 repo 裡的所有以下檔案：
- `SKILL.md`
- `CLAUDE.md`
- `AGENTS.md`
- `README.md`（僅限根目錄和直接子目錄的）

對每個掃描到的檔案，完整讀取內容。

---

## 第三步：逐份抽取

對每一份讀到的檔案，產出以下抽取卡片。一份檔案一張卡片，不得合併、不得跳過、不得摘要代替原文。

### 抽取卡片格式

```markdown
---
## [repo名稱] / [skill名稱]

### 來源
- repo：gstack / superpowers
- 路徑：（完整相對路徑）
- 檔案類型：SKILL.md / CLAUDE.md / AGENTS.md / README.md

### 一句話定位
（用繁體中文寫一句話：這個 skill 是做什麼的）

### 核心人格特質
（這個 skill 會讓 AI 表現出什麼樣的「性格」或「思考方式」？
例如：謹慎、挑戰性、發散性、結構化、懷疑論、使用者導向...
列出所有能辨識的特質，不限數量）

### 核心思考框架
（這個 skill 教 AI 用什麼框架去思考問題？
逐條列出，保留原文的關鍵句子或步驟。
如果原文有編號步驟，保留編號。
不要用自己的話改寫，盡量保留原始表述。）

### 核心行為規則
（這個 skill 要求 AI 做什麼 / 不做什麼？
分成「必須做」和「禁止做」兩區。
逐條列出，保留原文關鍵句。）

### 提問方式
（這個 skill 教 AI 怎麼問問題？
有沒有特定的提問模板、提問順序、提問策略？
如果有，完整列出。如果沒有，寫「無明確提問模板」。）

### 審查維度
（這個 skill 教 AI 從哪些角度審查一個方案或程式碼？
逐條列出所有審查維度。
如果不是審查型 skill，寫「非審查型」。）

### 輸出格式要求
（這個 skill 對 AI 的回覆格式有沒有特定要求？
例如：要分段、要用表格、要先總結再展開、要列優缺點...
如果有，完整列出。如果沒有，寫「無特定格式要求」。）

### 適用場景
（什麼時候應該用這個 skill？
盡量具體：是在規劃時？架構時？施工時？審查時？debug 時？
可以有多個場景。）

### 原文精華摘錄
（直接從原文中摘錄最重要的 5-15 句原文，不改寫。
這些句子是這個 skill 最核心的指導原則。
用 > 引用格式標記。）

### 和其他 skill 的潛在關聯
（讀完這份 skill 後，你覺得它和同 repo 或另一個 repo 的哪些 skill 有相似性或互補性？
只寫你看到的，不確定也列出來加問號。
格式：skill名稱（repo） - 相似/互補 - 簡述理由）

### 分類標記
（從以下選項中選出所有適用的，可多選）
- [ ] 思考框架型（影響 AI 怎麼想）
- [ ] 審查型（教 AI 從特定角度檢查）
- [ ] 工具程序型（教 AI 做特定操作的步驟）
- [ ] 人格型（賦予 AI 特定性格或態度）
- [ ] 流程型（定義一個完整工作流程）
- [ ] 元技能型（教 AI 怎麼使用其他 skill）
---
```

## 注意事項

1. **不要省略任何 skill**。即使看起來不重要或和本系統無關，也完整抽取。這是 raw data，判斷在下一輪做。
2. **不要用自己的話改寫原文精華摘錄**。直接引用原文。
3. **不要合併相似的 skill**。一份 SKILL.md 就是一張卡片，不例外。
4. **如果一個 skill 內容很少**（例如只有 3-5 行），卡片裡對應欄位寫「原文過短，無法提取」，但仍保留卡片。
5. **如果一個 skill 內容非常長**（例如超過 500 行），仍完整讀取。核心思考框架和行為規則要完整列出，原文精華摘錄可擴展到 20 句。
6. **「和其他 skill 的潛在關聯」這個欄位**：建議在所有卡片都初步填完後，再回頭統一補填這個欄位和跨 repo 相似性矩陣。因為要看完全部才能做好關聯判斷。
7. **語言**：一句話定位、適用場景用繁體中文。其他欄位如果原文是英文，保留英文原文。
